# Функциональные возможности Syt (для дизайн-промпта)

Трекер задач как Telegram Mini App: веб-приложение открывается по кнопке «Open» в боте (из списка чатов или из чата с ботом). Вход по Telegram, без отдельного логина.

---

## 1. Экраны и навигация

- **Главная (/**)**  
  - Приветствие, текущая дата (день недели, число, месяц).  
  - Если не в Telegram: карточка «Откройте приложение в Telegram» + кнопка «Открыть бота».  
  - Если в Telegram, загрузка: спиннер + «Вход через Telegram».  
  - Если ошибка входа: карточка с текстом ошибки + кнопка «Обновить страницу».  
  - Если пользователь ещё не подтянулся: карточка «Подключаем аккаунт в фоне» + кнопка «Открыть трекер сейчас».  
  - Если пользователь есть: приветствие по имени + кнопка «Перейти к задачам».  
  - Внизу ссылка «Экран в стиле трекера с градиентом →» (на трекер).

- **Трекер на день (/tracker)**  
  - Заголовок «Трекер на день».  
  - Блок выбора дня: аватар/иконка, переключатель «Пред. день / Сегодня [дата] / След. день», счётчик «0/0».  
  - Основная область: заглушка «На сегодня задач нет» + иконка (например уточка).  
  - Кнопка «Добавить задачу» (ведёт на создание задачи).  
  - Кнопка «Все задачи» (ведёт в список задач).

- **Список задач (/tasks)**  
  - Заголовок «Задачи», справа счётчик выполненных «N/M».  
  - Если не в Telegram и нет пользователя: карточка «Откройте в Telegram».  
  - Состояния загрузки, ошибки, пустой список, список с задачами.  
  - Список: карточки задач (название, статус, срок при наличии), чекбокс выполненности, кнопка удаления по ховеру.  
  - Внизу кнопка «+ Добавить задачу».

- **Новая задача (/tasks/new)**  
  - Кнопка «Назад», заголовок «Новая задача».  
  - Форма: поле «Название» (обязательно), «Описание» (необязательно), кнопка «Добавить задачу».  
  - При успехе — переход в список задач.

- **Повторения (/recurring)**  
  - Заголовок «Повторения».  
  - Пока заглушка: «Правила повторений появятся здесь».

- **Напоминания (/reminders)**  
  - Заголовок «Напоминания».  
  - Пока заглушка: «Скоро здесь появятся напоминания».

- **Общая оболочка (Layout)**  
  - Шапка: логотип/название «Syt», дата (на десктопе), навигация (Трекер, Задачи, Повторения, Напоминания), аватар/имя пользователя, кнопка «Закрыть» (в Telegram) или «Выход».  
  - Нижняя навигация (мобильная): те же 4 пункта — Трекер, Задачи, Повторения, Напоминания.  
  - Поддержка темы Telegram (фон из Telegram).

---

## 2. Задачи (сущность и действия)

- **Типы задачи:** задача (TASK), цель (GOAL), заметка (NOTE).  
- **Статусы:** активна (ACTIVE), выполнена (COMPLETED), в архиве (ARCHIVED).  
- **Поля:** название, описание (опционально), приоритет (LOW / MEDIUM / HIGH), срок (dueDate).  
- **Действия:** создать, список по пользователю, отметить выполненной/невыполненной (галочка), удалить (с подтверждением).  
- В карточке задачи отображаются: название, статус, срок (если есть); выполненная — зачёркнутый текст, заполненный чекбокс.

---

## 3. Повторяющиеся задачи (бэкенд есть, UI — заглушка)

- Правило повторения привязано к одной задаче-шаблону.  
- Частота: ежедневно, еженедельно (дни недели), ежемесячно (день месяца), свой интервал (CUSTOM).  
- Параметры: интервал, дни недели (1–7), дата окончания (опционально).  
- Система по крону создаёт экземпляры задач на день по правилам; при создании копируются напоминания шаблона.  
- Нужен UI: список правил повторений, создание/редактирование/удаление правила, отображение частоты и параметров.

---

## 4. Напоминания (бэкенд есть, UI — заглушка)

- Напоминание: привязка к задаче, дата/время срабатывания, флаг «отправлено».  
- Действия: создать, список по пользователю, пометить отправленным, удалить.  
- Крон раз в минуту помечает просроченные как отправленные (логика уведомлений в ТГ не реализована).  
- Нужен UI: список напоминаний, создание напоминания к задаче (выбор задачи + дата/время).

---

## 5. Пользователь и авторизация

- Вход только через Telegram (Mini App): пользователь определяется по данным Telegram, get-or-create по telegramId.  
- В приложении: отображаются имя (firstName), аватар (инициал). В модели есть: lastName, username, timezone, тариф FREE/PRO.  
- Нет отдельного экрана настроек профиля; кнопка «Закрыть» в шапке (в Telegram) закрывает Mini App.

---

## 6. Telegram-бот (чат с ботом)

- **/start** — регистрация/приветствие, инлайн-кнопка «ОТКРЫТЬ» (открывает Mini App).  
- **/tasks** — список задач (название + статус).  
- **/new** — пошаговое создание задачи: название → описание (или пропуск) → срок (или пропуск) → приоритет (LOW/MEDIUM/HIGH) → тип (TASK/GOAL/NOTE).  
- **/done &lt;taskId&gt;** — отметить задачу выполненной.  
- **/recurring &lt;taskId&gt;** — показать правило повторения для задачи.  
- **/remind &lt;taskId&gt; HH:MM** — создать напоминание на время (UTC).

---

## 7. Состояния и сценарии для дизайна

- Первый заход в Telegram: загрузка → приветствие → переход к задачам или трекеру.  
- Ошибка сети/сервера: сообщение об ошибке, кнопка «Обновить страницу».  
- Пустой список задач: заглушка + CTA «Добавить задачу».  
- Список с задачами: карточки, счётчик выполненных, переключение выполненности, удаление.  
- Создание задачи: форма с валидацией (название обязательно), после успеха — редирект в список.  
- Трекер на день: выбор дня (пока без привязки к данным), заглушка «На сегодня задач нет», CTA в «Задачи» и «Добавить задачу».  
- Повторения / Напоминания: пока только заголовок и заглушка (контент под будущий UI).  
- Открытие не из Telegram: одна карточка с призывом открыть в Telegram и кнопкой перехода к боту (без автоматического редиректа).

---

## 8. Технический контекст для дизайна

- Приложение — Telegram Mini App (открывается внутри Telegram, мобильные и десктопные клиенты).  
- Адаптивная вёрстка: верхняя навигация на широких экранах, нижняя панель на узких.  
- Тема: учёт фона Telegram (bg_color), иначе светлая тема (белый/светло-серый фон, индиго-акценты).  
- Компоненты: кнопки (primary/secondary), поля ввода, карточки, ссылки, спиннер загрузки, иконки (SVG).
