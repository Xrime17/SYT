# Как перенести дизайн из Figma в проект

Макеты Syt собраны в Figma (тёмная тема, компоненты, все основные экраны). Ниже — что в текущем фронте **оставить**, что **заменить**, и как подогнать всё под макеты.

> **Хочешь сразу сделать один экран?** → открой **[QUICK_START_FIGMA.md](./QUICK_START_FIGMA.md)** — там пошагово на примере Reminders и готовый код.

### Работа через MCP (Talk To Figma)

Когда плагин **Talk To Figma** подключён к Cursor:

1. **Выдели в Figma** нужный фрейм (например, «Reminders screen» или «Recurring tasks screen»).
2. **В чате напиши**, что переносим: например, «Перенеси этот экран в код» или «Выделен Reminders screen — сделай страницу».
3. Я запрошу структуру выделения через MCP (`read_my_design` / `get_selection`), по ней сгенерирую разметку и стили в нужный файл страницы.

То есть да: **достаточно выделить фрейм в Figma и написать в чате** — по выделению я получу данные и перенесу экран в код.

## Что оставить, что заменить

Текущий фронт **не выбрасывать целиком**. Ниже — что пригодится и что править.

### ✅ Оставить без изменений (логика и инфраструктура)

| Что | Зачем |
|-----|--------|
| **`_app.tsx`** | Провайдеры (Theme, User, SWR), TelegramUserLoader, шрифт Inter — всё нужно. |
| **`context/`** (UserContext, ThemeContext) | Состояние пользователя, темы, флаг «в Telegram» — не трогать. |
| **`api/`** (client, tasks, users, reminders, recurring) | Запросы и типы (Task, User и т.д.) — основа для данных. |
| **`hooks/useTelegramUser.ts`** | Работа с Telegram WebApp — оставить. |
| **`config.ts`**, **`types/`** | Конфиг и типы — оставить. |
| **Роутинг** | Маршруты `/`, `/tracker`, `/tasks`, `/tasks/new`, `/recurring`, `/reminders` — уже совпадают с макетами. |

### ✅ Оставить и только перестилизовать

| Что | Что сделать |
|-----|-------------|
| **`styles/globals.css`** | Токены `--syt-*` и тёмная тема в `.dark` **уже совпадают с Figma** (#0F1117, #1C2128 и т.д.). Оставить. Сделать тёмную тему по умолчанию (вешать `class="dark"` на `<html>` или всегда использовать тёмные токены). |
| **Компоненты** (Button, Card, Input, Textarea, Checkbox, TaskCard, EmptyStateCard, ErrorCard, Spinner, Avatar, NavTabs, BottomNav, PriorityBadge, StatusBadge) | Уже используют `var(--syt-*)`. **Оставить файлы и API (пропсы)**. При необходимости подправить вёрстку/отступы под макеты (например порядок полей в карточке, размеры). |
| **`Layout.tsx`** | Структура совпадает с Figma: шапка (Syt, дата, навигация, аватар, закрытие), нижняя навигация на мобиле, контент по центру. **Заменить только стили**: убрать светлые градиенты (indigo, slate, white), перевести на фон/текст/бордеры из токенов (`--syt-background`, `--syt-surface`, `--syt-text` и т.д.). |

### ⚠️ Заменить под макеты (вёрстка и копирайт)

| Что | Что сделать |
|-----|-------------|
| **Страницы** (`index.tsx`, `tracker.tsx`, `tasks/index.tsx`, `tasks/new.tsx`, `recurring.tsx`, `reminders.tsx`) | **Логику оставить**: useUser, useSWR, createTask, редиректы, обработчики. **Заменить**: разметку и тексты по Figma (заголовки, блоки, порядок кнопок, пустые/ошибочные состояния). Убрать светлые классы (gradients, slate), использовать компоненты дизайн-системы и токены. |
| **TaskItem vs TaskCard** | В коде есть и TaskItem, и TaskCard. Привести список задач к одному виду по макету «Tasks list» / «Visual task states» (чекбокс справа, заголовок слева, статус, дата, приоритет, удаление). Оставить один компонент (например TaskCard), подогнать под Figma. |

### Итог

- **Пригодится:** роутинг, API, контексты, хуки, конфиг, типы, **все токены в globals.css**, **все компоненты дизайн-системы** (как есть или с мелкими правками).
- **Менять:** только **внешний вид** — Layout и страницы перевести на тёмную тему и разметку из Figma, не переписывая запросы и состояние.

### С чего начать по шагам

1. **Включить тёмную тему по умолчанию** — в `ThemeContext` или в `_document`/Layout задать `document.documentElement.classList.add('dark')` (или всегда рендерить с `className="dark"`), чтобы везде применялись токены из `.dark` в `globals.css`.
2. **Перестилизовать Layout** — заменить классы в `Layout.tsx` на фон/текст/бордеры из `var(--syt-*)`, убрать градиенты и светлые цвета.
3. **По одной странице** — начать с Home (`index.tsx`), потом Tracker, Tasks list, Create Task, Recurring, Reminders; в каждой оставить логику, заменить разметку и тексты по Figma.

## 1. Figma как источник правды

- **Храните ссылку на файл** в одном месте (например, в README или в этом доке).
- При доработке UI сначала обновляйте макет в Figma, затем приводите в соответствие код.
- Для экспорта иконок/иллюстраций: в Figma выберите слой → правый клик → Export → PNG/SVG.

## 3. Как «добавить всё в проект»

Дизайн уже описан в коде через **дизайн-систему** (`docs/DESIGN_SYSTEM.md`). Задача — довести экраны и компоненты до макетов Figma.

### Шаг 1: Сверка с дизайн-системой

- Цвета, отступы, радиусы в коде должны совпадать с токенами из `DESIGN_SYSTEM.md` (они совпадают с Figma).
- Компоненты: Button, Input, Textarea, Checkbox, Card, TaskCard, EmptyStateCard, ErrorCard, Spinner, Avatar, NavTabs, BottomNav, PriorityBadge, StatusBadge.

### Шаг 2: Экраны по одному

Для каждого экрана:

1. Открыть нужный фрейм в Figma (например, «Tasks list screen»).
2. Сравнить структуру и тексты с соответствующей страницей (например, `pages/tasks/index.tsx`).
3. Доработать разметку и компоненты: заголовки, счётчики, карточки, кнопки, пустые/ошибочные состояния — как на макете.
4. Для списков (задачи, повторения, напоминания) использовать существующие компоненты (TaskCard, карточки правил/напоминаний), подставляя данные из API.

### Шаг 3: Задачи и карточки задач

- **Типы задач:** TASK, GOAL, NOTE (при необходимости добавить в бэкенд и в UI селектор/тип).
- **Статусы:** ACTIVE, COMPLETED, ARCHIVED — отображать через `StatusBadge`.
- **Приоритет:** LOW, MEDIUM, HIGH — через `PriorityBadge`.
- **Выполненная задача:** зачёркнутый заголовок (или класс с пониженной opacity), отмеченный чекбокс, приглушённый текст — как в фрейме «Visual task states».

### Шаг 4: Новые экраны/блоки по Figma

- **Recurring:** пустое состояние + список правил (карточка: название, частота, Edit/Delete) + кнопка «+ Add recurring rule».
- **Reminders:** пустое состояние + список напоминаний (задача, время, статус Scheduled/Sent, Delete) + форма «Create reminder» (Select task, date, time + кнопка).
- **Create Task:** форма с полями по макету (в т.ч. приоритет, тип задачи, опциональная дата), валидация «Title is required», после успеха — редирект в список задач.

### Шаг 5: Layout

- Подогнать общий layout под «Syt — Main layout»: шапка (Syt, дата, навигация, аватар, закрытие), на мобиле — нижняя навигация, контент по центру max-width 640px, отступы 16px.

## 4. Как переносить один экран (пошагово)

Делаете **вручную в коде**, глядя на Figma. Автогенерации кода из Figma в проекте нет — только референс.

### Алгоритм на примере Reminders

1. **Открыть в Figma** фрейм «Reminders screen» (как у вас сейчас).
2. **Открыть в редакторе** `frontend/src/pages/reminders.tsx`.
3. **Сверху вниз** повторить структуру с макета:
   - Заголовок «Reminders» → `<h1>` с классом по токенам (`text-[var(--syt-text)]`, `font-semibold`, `text-xl`).
   - Пустое состояние «Reminders will appear here soon» → компонент `EmptyStateCard` или `Card` с текстом.
   - Список напоминаний (если есть данные) → свои карточки: название задачи, время, бейдж Scheduled/Sent, кнопка Delete (стиль как в Figma — `Button variant="ghost"` или текст с `text-[var(--syt-error)]`).
   - Блок «Create reminder» → поля через `Input` (Select task, Select date, Select time) и кнопка `Button variant="primary"` «Create reminder».
4. **Стили** — только токены: `bg-[var(--syt-background)]`, `text-[var(--syt-text)]`, `border-[var(--syt-border)]`, отступы `p-4` (16px), `gap-6` (24px), как в auto-layout макета (padding 16, gap 24).
5. **Контейнер страницы** — как в Layout: контент в обёртке с `max-w-[640px] mx-auto px-4`, чтобы совпадало с Figma по ширине и отступам.

### Что брать из Figma, что — из кода

| В Figma смотрите | В коде используете |
|-------------------|--------------------|
| Порядок блоков, подписи кнопок, тексты пустых состояний | Разметку и копирайт |
| Цвета (#0F1117, #1C2128…) | Уже в токенах `globals.css` → классы `var(--syt-background)` и т.д. |
| Отступы (16, 24 px) | Tailwind: `p-4`, `gap-6`, `mb-6` |
| Кнопки, инпуты, карточки | Готовые компоненты из `@/components` или `@/components/design-system` |

### Один экран — один файл страницы

- **Reminders** → `pages/reminders.tsx`
- **Recurring** → `pages/recurring.tsx`
- **Tasks list** → `pages/tasks/index.tsx`
- **Create Task** → `pages/tasks/new.tsx`
- **Tracker** → `pages/tracker.tsx`
- **Home** → `pages/index.tsx`

Логику (запросы, состояние) не трогаете или добавляете по необходимости; меняете только JSX и классы под макет.

## 5. Экспорт из Figma (если нужно)

- **Иконки/иллюстрации:** выделить → Export → SVG или PNG 1x/2x.
- **Картинки для пустых состояний:** экспорт в `frontend/public/` или в папку статики и использование в `EmptyStateCard` и т.п.
- Код по макетам лучше писать вручную, опираясь на дизайн-систему: так остаётся один источник токенов и компонентов.

## 6. Чек-лист перед релизом

- [ ] Все экраны из Figma реализованы и визуально совпадают с макетами.
- [ ] Состояния (loading, error, empty, list) есть там, где они в Figma.
- [ ] Цвета и типографика через токены из `DESIGN_SYSTEM.md`.
- [ ] На мобиле отображается нижняя навигация, на десктопе — шапка с навигацией.
- [ ] Ссылка на Figma добавлена в README или в этот документ.

После этого всё, что сделано в Figma, будет учтено в проекте через дизайн-систему и пошаговую сверку экранов.
